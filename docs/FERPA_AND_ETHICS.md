# FERPA and Ethics Notes

This MVP must use fictional or de-identified data only.

The tool is designed to identify pathway barriers and improve institutional systems, not to label students as likely to fail.

## Approved MVP use

- Analyze degree-map fragility.
- Identify bottleneck courses.
- Compare course-sequence options.
- Support structured scheduling decisions.
- Support advisor conversations with human review.
- Test hypothetical schedule improvements.

## Prohibited use

- Do not use real student-level records in the MVP.
- Do not create a student risk score.
- Do not label students as likely failures or dropouts.
- Do not automate denial of courses, services, supports, programs, or opportunities.
- Do not make high-stakes decisions without human review.

## Future production requirements

Before using real data, production versions must include:

- FERPA review,
- institutional data governance approval,
- role-based access control,
- data minimization,
- audit logs,
- secure hosting,
- bias testing,
- human-in-the-loop advising review,
- explainable recommendation logic,
- appeal/review process for any recommendation affecting a student.

## Required app notice
Add this visible notice to the MVP:

> This MVP uses fictional/demo data only. It is designed to support pathway design, advising strategy, and schedule planning. It should not be used to make high-stakes decisions about individual students.
